# Начало работы с DNS-хостингом

Чтобы сайт открывался по доменному имени, а почта приходила на нужные ящики, домен нужно "связать" с сервером - именно за это отвечают DNS-записи. Они превращают доменные имена вроде `example.com` в IP-адрес сервера, на котором живёт ваш проект.

В этом гайде мы разберёмся, как делегировать домен на наши серверы имён (DNS серверы) и управлять DNS-записями через панель **DNSmanager** хостинга Senko Digital.

---

## Что вам понадобится

Перед началом убедитесь, что у вас есть всё необходимое:

- **Доменное имя** (`.com`, `.org`, `.net` или любое другое) - если домена ещё нет, можете [зарегистрировать его у нас](https://senko.digital/domains)
- **Активный виртуальный или выделенный сервер** в Senko Digital - именно к его IP-адресу мы будем привязывать домен
- **Доступ к панели DNSmanager** - данные для входа приходят в письме об активации услуги DNS хостинга

::: tip
DNSmanager доступен бесплатно каждому клиенту - даже тем, у кого нет активных услуг.
:::

## Делегирование домена на наши серверы имён

Прежде чем создавать записи, домен нужно делегировать на наши NS, иначе DNS-записи, которые вы создадите в панели, попросту не будут работать.

Укажите у регистратора вашего домена следующие серверы имён:

```text
ns1.senkodns.net
ns2.senkodns.net
```

Где именно поменять NS - зависит от регистратора. Если домен зарегистрирован у нас, делегирование настраивается в параметрах домена через личный кабинет. Если у стороннего регистратора - ищите раздел **"Серверы имён"**, **"NS-серверы"** или **"DNS servers"** в панели управления доменом.

::: warning Важно
После смены NS-серверов изменения проходят не мгновенно - на это может уйти **до 72 часов**. Чаще всего домен начинает работать гораздо раньше, например в течение нескольких часов.
:::

## Вход в панель DNSmanager

1. Откройте в браузере [https://dns.senko.digital](https://dns.senko.digital).
2. Введите **логин** и **пароль** из письма об активации услуги DNS-хостинга.
3. Нажмите **"Войти"**.

После авторизации вы окажетесь на главной странице DNSmanager - отсюда мы и будем управлять доменами.

## Создание домена в DNSmanager

Делегирование - это только лишь половина дела. Чтобы наши серверы имён начали обрабатывать запросы к вашему домену, его нужно завести и в самой панели.

1. Перейдите в раздел **"Главное" → "Доменные имена"**.
2. Нажмите кнопку **"Создать"** в верхней части окна.
3. В открывшейся форме заполните поля:

   - **Тип** - обязательно выберите `master`. Без этого записи работать не будут.
   - **Имя** - ваше доменное имя (например, `example.com`).
   - **IP-адрес** - IP сервера, на котором размещён (или будет размещаться) сайт.
   - **Email администратора** - контактный адрес администратора домена, обычно стоит указать личный email.

4. Нажмите **"Ok"** для сохранения.

Если всё прошло успешно, домен появится в списке в разделе **"Доменные имена"**, а DNSmanager автоматически создаст для него стандартный набор базовых записей (`SOA`, `NS`, `A`).

::: tip
После создания домена остаётся только дождаться обновления глобального кэша DNS. Это та же история, что и с делегированием - обычно несколько часов, а в худшем случае - до 72 часов.
:::

## Управление DNS-записями

Чтобы посмотреть или изменить записи уже добавленного домена:

1. Откройте раздел **"Главное" → "Доменные имена"**.
2. Выделите нужный домен в списке.
3. Нажмите кнопку **"Записи"** в верхней панели.

В открывшемся окне будут все записи выбранного домена. Здесь же доступны кнопки **"Создать"**, **"Изменить"** и **"Удалить"** - ими и редактируется DNS-зона.

### Пример: создание A-записи для поддомена

Разберём на практике, как добавить A-запись для поддомена `server.example.com`, чтобы он указывал на нужный IP-адрес.

1. В окне записей домена нажмите **"Создать"**.
2. Заполните поля:

   - **Имя** - имя поддомена. Можно указать просто `server` (доменное имя добавится автоматически) или `server.example.com.` целиком - но тогда **обязательно с точкой в конце**.
   - **TTL** - время жизни записи. Оставьте значение по умолчанию, если не знаете, для чего это нужно.
   - **Тип** - выберите `A`.
   - **IP-адрес** - IP сервера, на который должна указывать запись.

3. Нажмите **"Ok"**.

::: warning Важно
Точка в конце полного доменного имени - не опечатка, а важная деталь синтаксиса DNS. Без неё панель припишет к вашей записи имя домена ещё раз и вместо `server.example.com` получится `server.example.com.example.com`.
:::

После сохранения новая запись появится в общем списке. Готово - поддомен начнёт открываться, как только обновится кэш DNS.

## Типы DNS-записей

Принцип создания всех записей одинаковый - отличаются только поля, которые нужно заполнить. Вот наиболее частые типы записей и для чего они нужны:

| Тип       | Для чего нужна                                                                 |
| --------- | ------------------------------------------------------------------------------ |
| **A**     | Привязывает домен или поддомен к IPv4-адресу сервера.                          |
| **AAAA**  | То же самое, но для IPv6 адресов.                                               |
| **CNAME** | Алиас - указывает, что один домен является "псевдонимом" другого.              |
| **MX**    | Маршрутизация почты - на какой сервер отправлять письма для вашего домена.     |
| **TXT**   | Обычный текст. Часто используется для SPF, DKIM, DMARC и верификации на различных ресурсах (например Google Workspace).    |
| **CAA**   | Указывает, какие удостоверяющие центры могут выпускать SSL-сертификаты домена. |
| **NS**    | Делегирует поддомен на отдельные серверы имён.                                 |
| **SRV**   | Используется некоторыми сервисами (Matrix, XMPP, Minecraft и др.).             |

::: tip
Если только что заведённый домен ещё не открывается - это вполне нормальное явление. Подождите немного и попробуйте снова. Для уже существующих доменов новые записи обычно подхватываются гораздо быстрее, чем самое первое делегирование.
:::

## Что делать, если ничего не работает

Если после настройки домен или поддомен всё ещё не открывается, чаще всего дело в одном из этих пунктов:

- **NS-серверы у регистратора не были изменены на наши** - проверьте, что там указаны именно `ns1.senkodns.net` и `ns2.senkodns.net`.
- **Не прошли 72 часа с момента изменений** - DNS-записям нужно время на распространение по миру.
- **В A-записи указан не тот IP** - убедитесь, что IP действительно принадлежит серверу с вашим сайтом.
- **В полном доменном имени забыта точка в конце** - без неё запись "склеивается" и выходит неправильное имя.
- **Тип домена в DNSmanager выставлен не в `master`** - в этом случае записи не будут отдаваться нашими серверами имён.

Проверить, как ваш домен "видит" интернет, можно с помощью утилиты `dig` или онлайн-сервисов вроде [dnschecker.org](https://dnschecker.org). Если NS-серверы там показываются правильно, но домен всё равно не открывается - скорее всего, дело в кэше провайдера, и достаточно подождать.

### Очистите локальный кэш DNS

Если различные сервисы уже показывают новые записи, а у вас на компьютере домен по-прежнему открывается со старыми данными (или не открывается вовсе) - скорее всего, виноват локальный DNS-кэш операционной системы. Очистить его можно одной командой:

- **Windows** (в командной строке от имени администратора):

  ```bash
  ipconfig /flushdns
  ```

- **macOS** (в терминале):

  ```bash
  sudo dscacheutil -flushcache && sudo killall -HUP mDNSResponder
  ```

- **Linux** (зависит от дистрибутива - чаще всего подойдёт одна из команд):

  ```bash
  sudo resolvectl flush-caches
  sudo systemd-resolve --flush-caches
  ```

После очистки кэша попробуйте открыть домен заново.

::: tip
Самый быстрый способ сразу обойти кэш и провайдера, и системы - открыть сайт со смартфона по мобильному интернету. Если там домен открывается корректно, дело точно в кэше где-то по пути, а не в самих записях.
:::
